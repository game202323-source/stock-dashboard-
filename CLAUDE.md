# stock-dashboard- 작업 지침

이 저장소는 AI반도체·헬스케어·에너지 밸류체인 39개 종목을 추적하는 React 대시보드입니다.
사용자가 "오늘자 업데이트해줘" 라고 요청하면 아래 절차를 그대로 따르세요.

## 1. 자동 루틴과 중복 작업 방지

`매일 시장 코멘트 업데이트` 라는 예약 루틴(Routine, cron 매일 00:10 UTC)이 이미 존재합니다.
이 루틴은 웹에서 반도체·헬스케어·에너지 관련 뉴스를 검색해 `src/data/marketComments.json`
배열 맨 앞에 오늘 날짜 항목을 추가(또는 같은 날짜가 있으면 교체)하고 **`main`에 직접 커밋·푸시**합니다.

**따라서 "오늘자 업데이트" 요청을 받으면 가장 먼저:**
```
git fetch origin main
git show origin/main:src/data/marketComments.json | head -50
```
으로 오늘 날짜 항목이 이미 main에 있는지 확인하세요. 이미 있다면 **직접 다시 만들지 말고**
그 내용을 그대로 사용해 아래 2번(대시보드)만 진행하면 됩니다. 없을 때만 직접 작성해서
PR을 올리세요 (그마저도 루틴이 먼저 커밋할 수 있으니, 작업 전에 한 번 더 main을 fetch해서
확인 후 진행). 과거에 이 확인을 생략해서 자동 루틴과 내용이 다른 중복 PR을 만든 적이
있으니 주의하세요.

**보관 기간: `marketComments.json` 배열은 항상 최신 2일치(당일 + 전날)만 유지합니다.**
사용자가 과거 기록이 계속 쌓인다고 반복해서 지적한 적이 있습니다 — 예약 루틴 프롬프트에
날짜 항목 추가/교체 로직만 있고 정리(trim) 로직이 없어서 매번 이 문제가 재발했었습니다.
"오늘자 업데이트" 작업 중 `marketComments.json`을 만지게 되면(직접 작성하든, 확인만 하든)
날짜 내림차순 정렬 후 **배열을 최신 2개 항목으로 잘라내고** main에 반영하세요. 3일치 이상
남아있는 것을 발견하면, 그 자체가 버그이니 별도 요청이 없어도 2일치로 정리해서 커밋하세요.

## 2. "대시보드 보여줘" 요청 시 — 반드시 실제 앱과 동일한 디자인으로

사용자는 커스텀 디자인이 아니라 **실제 배포된 앱(src/App.jsx + src/App.css)과 똑같이 생긴**
정적 스냅샷을 Artifact로 원합니다. 매번 새로 디자인하지 말고 다음을 그대로 재현하세요:

- 헤더: "AI 반도체 · 헬스케어 · 에너지 밸류체인 대시보드" + 부제
- "오늘의 시장 코멘트" 섹션: `marketComments.json`(main 기준) 내용을 그대로, 날짜 내림차순,
  파란 날짜/섹션 제목 + 불릿 리스트 스타일 (App.css의 `.comment-*` 클래스 그대로 사용)
- 섹터 탭(전체/AI반도체/장비/데이터센터/광통신/헬스케어/바이오/에너지)으로 필터링되는
  종목 카드 그리드. 카드마다 현재가/목표가 막대바 + 배지("여력 X%" 또는 "목표가 초과 +X%").
  배지 계산: `diffPct = (target - current) / current * 100`, `diffPct <= 0`이면 초과.
- `App.css`의 색상·spacing 그대로: 배경 `#f9fafb`, 카드 `#fff`/`#e5e7eb` 테두리,
  현재가 바 `#111827`(검정), 목표가 바 `#dc2626`(빨강), 여력 배지 `#dcfce7`/`#15803d`,
  초과 배지 `#fee2e2`/`#b91c1c`.

전체 39개 종목 목록은 `api/stocks.js`의 `TICKERS` 배열 참고 (이름/섹터/티커).

## 3. 종목 시세는 반드시 39개 전부, 실제 조사값으로

이 샌드박스 환경에서는 Yahoo Finance API(query1.finance.yahoo.com)가 403으로 막혀 있어
`api/stocks.js`처럼 직접 fetch할 수 없습니다. 대신 **WebSearch로 개별 조사**하세요:
- 섹터별로 7개 서브에이전트(Agent 툴, general-purpose)를 병렬로 띄워 각 섹터의 티커들을
  `"<TICKER> stock price analyst price target"` 등으로 검색해 현재가·목표주가(평균)를 가져오게 하세요.
- 스크린샷이나 과거 값으로 대충 채우지 말 것 — "예시"였던 적이 있었으니 전체 39개를 다 채우세요.

## 4. 종목마다 "밸류에이션 체크" 코멘트 필수

사용자는 "여력 54%" 같은 숫자만 보면 바로 사야 할 것 같은 착각을 하게 된다는 점을 지적했습니다.
각 종목 카드마다 2-3문장짜리 한국어 코멘트를 넣어서, 목표주가 괴리가 진짜 저평가인지
아니면 이미 잘 알려진 성장 스토리가 선반영된 것인지를 짚어주세요. 근거로 쓸 수 있는 정보
(선행 PER, 매출/이익 성장률, 최근 뉴스, 리스크 요인)도 서브에이전트 조사 단계에서 같이
수집하게 하세요. 이 코멘트는 투자 조언이 아니라 정보 제공이라는 점을 스냅샷 안내문과
footer에 명시하세요.
